# Skill: Foreshadow Check

떡밥(Plant)과 회수(Payoff)의 레이어 구조를 검증한다.

## 원칙

진격의거인식 떡밥 레이어:
**같은 장면/대사가 정보가 쌓일수록 의미가 완전히 바뀌어야 한다.**

## 검증 방법

### 1. Plant 목록 추출
각 에피소드의 떡밥 테이블에서 Plant를 추출한다.

### 2. Payoff 추적
각 Plant가 어디서 회수되는지 추적한다.

### 3. 레이어 깊이 확인
각 떡밥의 의미 변환 횟수를 확인한다.

| 깊이 | 판정 |
|------|------|
| 1회 (심고 회수) | 최소 — 개선 필요 |
| 2회 (심고 재해석 후 회수) | 양호 |
| 3회 이상 (심고 → 1차 재해석 → 2차 재해석 → 최종 회수) | 우수 |

### 4. 출력 형식

```markdown
## 떡밥: [이름]
- **심기 (X화)**: [상황] → 플레이어 인식: [?]
- **1차 재해석 (Y화)**: [새 정보] → 의미 변화: [?]
- **2차 재해석 (Z화)**: [새 정보] → 의미 변화: [?]
- **최종 회수 (W화)**: [확정] → 최종 의미: [!]
- **레이어 깊이**: N단계
- **판정**: 최소 / 양호 / 우수
```

### 5. 경고 조건
- Plant만 있고 Payoff 없음 → **미회수 경고**
- Payoff가 Plant보다 앞에 있음 → **순서 오류**
- 레이어 1단계만 있음 → **깊이 부족 경고**
- 핵심 떡밥(리본, 상처, 칼리스 등)이 3단계 미만 → **보강 필요**
