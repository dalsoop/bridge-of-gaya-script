# Skill: Visual Consistency

캐릭터 비주얼 일관성을 검증하고 유지한다.

## 일관성 체크 항목

### 1. 불변 요소 (절대 변하면 안 됨)

| 항목 | 설명 |
|------|------|
| 체형/키 | 에피소드 간 동일 |
| 눈 색 | 항상 동일 |
| 얼굴형 | 항상 동일 |
| 의상 기본형 | 스토리 변화 외 동일 |
| 무기 디자인 | 항상 동일 |
| 소품 디자인 | 리본 모양/색 등 항상 동일 |

### 2. 스토리 기반 변화 (허용되지만 추적 필요)

| 캐릭터 | 변화 | 트리거 | 복귀 여부 |
|--------|------|--------|-----------|
| 가야 | 검은 머리 → 은색 끝 | 15화 | 비복귀 (진행) |
| 가야 | 은색 끝 → 백발 | 23화 | 비복귀 |
| 가야 | 몸에 상처 추가 | 5화~ 매 루프 | 루프 리셋 시 일부 감소 |
| 가야 | 눈 밑 그림자 심화 | 루프 후반 | 루프 리셋 시 약간 감소 |
| 테이 | 갈색 머리 → 흰색 시작 | 20화 | 비복귀 |
| 칼리스 | 점점 반투명 | 22화~ | 23화 소멸 |

### 3. 씬별 변동 (일시적)

| 항목 | 변동 조건 | 기본으로 복귀 |
|------|-----------|---------------|
| 표정 | 매 씬마다 | 다음 씬 |
| 포즈 | 매 씬마다 | 다음 씬 |
| 조명/색감 | 시간대/분위기 | 다음 씬 |
| 의상 손상 | 전투 후 | 다음 날 |

## 검증 프로세스

### 프롬프트 작성 시
```
1. character-sheet에서 기본 프로필 로드
2. 현재 에피소드의 변형 적용 (있으면)
3. 불변 요소가 빠지지 않았는지 확인
4. 변화 요소가 올바른 트리거 이후인지 확인
5. 네거티브에 불필요한 변형 방지 태그 추가
```

### 생성된 이미지 검증 시
```
체크리스트:
- [ ] 머리색이 현재 에피소드 기준과 일치하는가
- [ ] 눈 색이 프로필과 일치하는가
- [ ] 의상이 기본형 또는 허용된 변형인가
- [ ] 소품(리본 등)이 현재 소지 상태와 일치하는가
- [ ] 상처/멍이 현재 루프 진행도와 일치하는가
- [ ] 체형/키가 다른 이미지와 일치하는가
- [ ] 무기 디자인이 동일한가
```

## 에피소드별 비주얼 상태 추적

```markdown
## EP.XX 비주얼 상태

### 가야
- 머리: [검정 | 은색 끝 | 백발]
- 리본: [있음 | 없음(리셋) | 피 묻음]
- 상처: [없음 | 미세한 멍 | 선명한 자상(위치: __)]
- 피로: [정상 | 약간 | 심함(눈 밑 그림자)]
- 표정 기조: [평온 | 불안 | 공포 | 집착 | 명확(본체)]

### 테이
- 머리: [갈색 | 일부 흰색 | ]
- 상처: [없음 | ]
- 표정 기조: [과묵 | 의심 | 충격 | 결의]
```

## 비주얼 떡밥 추적

비주얼 요소도 떡밥 레이어로 기능한다. foreshadow-layers.md와 연동.

| 비주얼 요소 | 첫 등장 | 의미 변화 | 최종 |
|-------------|---------|-----------|------|
| 붉은 리본 | 3화(선물) | 7화(피 묻음) | 24화(죽음 로그) |
| 가야 상처 | 5화(미세) | 12화(일치 확인) | 24화(칼자국) |
| 은색 머리끝 | 15화 | 20화(테이도) | 23화(백발) |
| 몬스터 실루엣 | 11화(인간형) | 19화(가야 겹침) | — |
| 프롤로그 구도 | 0화 | 25화(리플레이) | 27화(재현) |
