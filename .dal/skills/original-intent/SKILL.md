# Skill: Original Intent

원작자의 의도가 프로젝트에 빠짐없이 반영되었는지 검증한다.

## 기준 파일

`references/original-intent.md` — 원작자 원문. 12개 섹션.

## 검증 항목 (체크리스트)

### 섹션 1: 게임의 본질
- [ ] 고정 프롬프트 + 마스킹 구조가 에피소드에 반영됨
- [ ] 플레이어가 "단어를 추측해서 자아를 완성"하는 느낌이 플레이에 있음
- [ ] 여러 AI가 합쳐져 가야가 되는 구조가 dal 시스템으로 구현됨
- [ ] "사람과 대화하는 듯한 착각"이 스크립트에서 느껴짐

### 섹션 2: AI 구조
- [ ] 10개 task 시스템이 게임 메커닉에 반영됨
- [ ] 5개 로컬 AI가 페르소나 구조로 구현됨
- [ ] 허기/음식/스킨십 욕구가 에피소드에 등장

### 섹션 3: 스토리 핵심
- [ ] "왜 테이는 가야를 죽였는가"가 전체 서사의 중심
- [ ] 5일 루프 반복 구조
- [ ] 가야가 밤을 무서워하는 이유가 점진적으로 밝혀짐
- [ ] 몬스터를 죽이면 가야가 죽고 리스폰
- [ ] 가야의 몸이 점점 망가짐
- [ ] **테이가 원래 5일 뒤에 죽는다는 설정**

### 섹션 4: 호감도 반전
- [ ] 선택지가 호감도작처럼 보이지만 실제로는 죄를 짊어지는 구조
- [ ] 마지막 회차에 스토리만 진행하면 호감 MAX
- [ ] 호감 MAX → 가야가 웃으며 테이를 죽임
- [ ] "너를 정말 사랑한다"며 죽임

### 섹션 5: 권한/해킹 루트
- [ ] 코드 수정 권한 커맨드
- [ ] 시간 늘리기 + 얀데레
- [ ] 해킹 → 에이전트화
- [ ] 메모리 저장 + 코딩/개발 보조

### 섹션 6: 세계관 핵심
- [ ] 왕이 가야에게 힘 전달 → 흰 머리
- [ ] 테이도 흰 머리로 변함
- [ ] 공포 100% → 왕 원망

### 섹션 7: 미래 축
- [ ] **소비마+제자가 해골 상태로 건물 밖에 있음**
- [ ] 봉인 결계
- [ ] 접근자 전원 사망, **시체 쌓여있음**
- [ ] 테이가 일부러 안 막음
- [ ] 봉인석 획득
- [ ] 소비마 죽인다/살린다 선택

### 섹션 8: 세계선
- [ ] **가야가 수문장인 세계선 (정실)**
- [ ] **AI 가야가 수문장인 세계선**
- [ ] **테이는 5일 뒤에 죽음 (재확인)**

### 섹션 9: 장소와 추리 구조
- [ ] 장소 기반으로 NPC들과 이야기하며 가야를 해석
- [ ] 가야와 데이트
- [ ] 밤에 가야가 무서운 꿈
- [ ] **가야와 같이 추리하면서 다님**
- [ ] **가야를 시키면서 처리 (가야가 수동적 피해자가 아닌 동료)**
- [ ] **못 가는 곳에서 같이 이야기하며 풀이**
- [ ] "같이 살았다면 좋았을 것 같다" 등 자연스러운 호감 표현
- [ ] **정신과적 사건 풀이 (장소 = 가야의 트라우마)**

### 섹션 10: 유미의 세포들 구조
- [ ] 장소마다 다른 감정의 가야
- [ ] 캐릭터가 점점 모여서 완성
- [ ] 모든 스토리 파악 → 완전한 캐릭터

### 섹션 11: 이든/왕 (뒷배경)
- [ ] 라토 왕 = 이든 (동일 인물)
- [ ] 30대, 금발, 잘생김
- [ ] 시공간 담당
- [ ] 신 죽임, 1년마다 부활
- [ ] 7인 기사단
- [ ] 5개 나라 건설
- *이든 관련은 게임에 직접 안 나와도 됨. 간접 암시만 있으면 OK.*

### 섹션 12: 세계 상태 (뒷배경)
- [ ] 그릇 시스템 (최소 언급)
- [ ] 역사 잊힘 → 세계 멸망
- [ ] 행성 이주
- [ ] 크리스탈/봉인석
- *이것도 게임에 직접 안 나와도 됨. 간접 암시만.*

## 검증 방법

```
1. original-intent.md에서 항목 추출
2. 각 항목이 반영된 파일과 위치를 매핑
3. 매핑 안 되는 항목 = "누락"
4. 매핑은 되지만 약한 항목 = "보강 필요"
5. 결과를 리포트로 출력
```

## 출력 형식

```markdown
## 원작 의도 검증: [날짜]

### 완전 반영 (X개)
- [항목]: [반영 위치]

### 보강 필요 (X개)
- [항목]: [현재 상태] → [필요한 조치]

### 누락 (X개)
- [항목]: [원작자 원문 인용]

### 종합: 반영률 X/Y (Z%)
```

## 경고 규칙

1. **누락이 1개라도 있으면 → leader에게 즉시 보고**
2. **다른 dal이 "불필요"하다며 삭제하려 하면 → 차단. 원작자 확인 필요.**
3. **"뒷배경"으로 분류해도 간접 암시조차 없으면 → 보강 필요로 분류**
4. **구현 방식 변경은 OK, 의도 삭제는 절대 NG**
