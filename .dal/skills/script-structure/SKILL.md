# Skill: Script Structure

> 출처: scriptwriter-skill (MCP Market) 기반, VN 프로젝트 맞춤 변환

에피소드 스크립트의 구조적 완성도를 보장한다.

## 핵심 원칙

### 1. 시각 우선 (Visual First)
- 추상적 심리 묘사 대신 **구체적 시각 묘사** 사용
- ❌ "가야는 불안했다"
- ✅ "가야의 손이 떨리고 있었다. 시선이 바닥에 고정되어 있었다."
- 모든 감정은 **행동/외형/반응**으로 보여줘야 함

### 2. 캐릭터 일관성
- 첫 등장 시 외형 묘사 필수
- 핵심 식별자 유지 (가야=검은 머리, 칼리스=날카로운 눈빛 등)
- 성격 변화는 점진적. 급변 시 전환 씬 필수.

### 3. 3막 구조 (에피소드 단위)
각 에피소드 내에서도 미니 3막을 유지:

| 막 | 비중 | 역할 |
|----|------|------|
| 1막 (Setup) | ~25% | 상황 제시, 이전 화 연결 |
| 2막 (Confrontation) | ~50% | 핵심 이벤트, 갈등, 발견 |
| 3막 (Resolution) | ~25% | 결과, 다음 화 훅 |

### 4. 비트 최소 기준
에피소드당 **최소 5개의 명확한 전환점(비트)**:

```
비트 1: 상황 변화 (뭔가 달라짐)
비트 2: 반응 (캐릭터가 대응)
비트 3: 심화 (더 깊어짐)
비트 4: 전환 (방향 바뀜)
비트 5: 훅 (다음으로 당김)
```

핵심 화(7, 12, 19, 24, 27화)는 **최소 7비트**.

## 시리즈 연속성 체크리스트

에피소드 작성 전 반드시 확인:

- [ ] 이전 에피소드의 마지막 상태 (가야 감정, 테이 상태, 장소)
- [ ] 현재 루프 회차 / Day
- [ ] 활성 떡밥 목록 (미회수 Plant)
- [ ] 캐릭터 외형 변화 (상처, 머리색 등)
- [ ] 열린 질문 스택 (mystery-cascade)
- [ ] 기사단 NPC 마지막 대사

## 에피소드 완성 기준

- [ ] 씬 헤더 전부 작성 (장소, 시간, BGM, 캐릭터)
- [ ] 대사에 표정 태그 포함
- [ ] 선택지에 flags 명시
- [ ] 비트 5개 이상 확인
- [ ] 3막 비중 균형 확인
- [ ] 시각적 묘사 > 심리적 서술
- [ ] 다음 화 훅 존재
