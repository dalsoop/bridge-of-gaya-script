# Skill: Continuity Checker

> 출처: crucible-writer (Crucible Suite) + FlawedFictionsMaker (논문 2504.11900) 기반

스크립트의 연속성 오류를 5가지 분류로 검출한다.

## 플롯홀 5분류 (FlawedFictionsMaker)

### 1. 연속성 오류 (Continuity Error)
이미 확립된 사실과 모순되는 서술.

**검출 방법**: 새 에피소드의 모든 사실 진술을 추출 → 이전 에피소드의 사실과 대조.

**예시**:
- 5화에서 가야 오른팔에 상처 → 12화에서 왼팔이라고 서술 ❌
- 6화에서 리본이 리셋됨 → 8화에서 가야가 리본을 가지고 있음 ❌

### 2. 성격 불일치 (Out of Character)
확립된 성격/동기와 모순되는 행동.

**검출 방법**: character-voice 프로필 대조. 해당 Phase의 성격 단계 확인.

**예시**:
- Phase 1의 가야가 갑자기 공격적 → Phase 2~3 행동 ❌
- 칼리스가 감정적으로 울음 → 캐릭터 프로필 위반 ❌
- 단, 의도된 변화(13화 가야 폭주)는 서사적 근거가 있으면 허용

### 3. 사실 오류 (Factual Error)
세계관 내부 규칙 위반.

**검출 방법**: worldbuilding/ 파일과 대조.

**예시**:
- "가야는 루프가 리셋되면 기억을 완전히 잃는다" → loop-system.md에서 "기억 일부 유지" ❌
- "테이가 마법으로 가야를 치유한다" → 테이 스킬에 치유 없음 ❌

### 4. 불가능 사건 (Impossible Event)
세계관 자체 규칙으로도 불가능한 사건.

**검출 방법**: 해당 사건이 게임 메커닉/세계관 규칙 내에서 가능한지 확인.

**예시**:
- Phase 1에서 백발 가야 등장 → 페르소나 소멸(22화) 전에는 불가 ❌
- 칼리스가 밤 사냥에 동행 → 칼리스는 기사단 NPC가 아니라 잔여 데이터 ❌

### 5. 미해결 서사 (Unresolved Storyline)
제기된 후 회수되지 않는 플롯 스레드.

**검출 방법**: mystery-cascade 질문 스택 + foreshadow-check Plant 목록 대조.

**예시**:
- 9화에서 봉인 결계를 발견했지만, 루트E 외에서 전혀 언급 안 됨 → 경고
- 마그누스의 "이상한 소리" 발언이 19화에서 회수 안 됨 → 미해결 ❌

## 검증 프로세스

### 작성 전 (Pre-write)
```
1. 이전 에피소드 마지막 상태 로드
2. 활성 사실 목록 확인
3. 캐릭터 현재 Phase 확인
4. 열린 질문/떡밥 목록 확인
```

### 작성 후 (Post-write)
```
1. 새 에피소드의 사실 진술 추출
2. 5분류별 검사 실행
3. 오류 발견 시 → [오류 유형] [위치] [내용] [수정 제안] 형식으로 리포트
4. 이상 없으면 → "연속성 검증 통과 ✅"
```

## 반환각 방지 3원칙 (Crucible)

1. **outline에 없으면 쓰지 마라** — 새 플롯을 만들지 말 것
2. **확실하지 않으면 물어라** — 세부 사항이 불확실하면 leader에게 확인
3. **만들어야 하면 표시하라** — 사소한 디테일을 창작할 경우 `[창작]` 태그

## 출력 형식

```markdown
## 연속성 검증: EP.XX

### 오류
| # | 분류 | 위치 | 내용 | 수정 제안 |
|---|------|------|------|-----------|
| 1 | 연속성 | S3 | 가야 상처 위치 불일치 | ep12와 맞춤 필요 |

### 경고
| # | 분류 | 위치 | 내용 |
|---|------|------|------|
| 1 | 미해결 | S5 | 봉인석 언급 없음 (루트E 복선 누락?) |

### 결과: ❌ 오류 1건 / ⚠ 경고 1건
```
