# Skill: Mystery Cascade

"하나를 답하면 하나를 새로 던진다" — 질문 스택 관리 및 검증.

## 핵심 원칙: Reveal Cascade

미스터리에서 각 주요 씬은:
1. **기존 질문 하나를 답한다**
2. **새로운, 더 큰 질문을 하나 던진다**

이렇게 질문 스택이 계속 갱신되며, 최종 화에서 전부 해소된다.

## 질문 스택 관리

```yaml
question:
  id: "Q01"
  text: "가야는 왜 밤을 무서워하는가?"
  raised_in: "ep01_s6"
  partial_answers:
    - scene: "ep05_s2"
      answer: "몸에 상처가 생김"
      new_question: "Q04"
    - scene: "ep12_s2"
      answer: "상처가 사냥 부위와 일치"
      new_question: "Q07"
  final_answer:
    scene: "ep19_s2"
    answer: "밤 사냥 = 가야의 죽음"
  status: resolved
```

## 검증 규칙

### 1. 질문 고갈 금지
질문 스택이 **비는 순간이 없어야** 한다.
답변으로 질문이 줄어도, 새 질문이 추가되어 총량 유지.

### 2. 질문 과부하 금지
미해결 질문이 **동시에 7개 이상**이면 플레이어가 혼란.
→ 일부를 부분 답변으로 정리.

### 3. 공정한 미스터리
답변 시점에서 **플레이어가 추론 가능한 단서가 이미 제공되어 있어야** 함.
→ 답변과 함께 "이 단서들로 알 수 있었다" 역추적 가능해야.

### 4. 질문 깊이
표면 질문(Q01~Q04)은 중반에 해소.
핵심 질문(Q10)은 최종 Phase에서만 해소.
질문이 깊어질수록 답변 시점이 늦어져야 함.

### 5. 최종 화 질문 잔량
27화(분기점) 시점에서 미해결 질문은 **Q10, Q11만** 남아야.
나머지는 전부 해소.

```markdown
| Q04 (상처 원인) | Q07 (사냥과 가야 관계) |

```
