# AI 구조 / 마스킹 / Task 시스템

## 마스킹된 가야 (핵심 정의)

- 가야 본체 = **고정된 프롬프트**
- 일부 핵심 정보(키워드/성향/기억)가 **마스킹**됨
- LLM은 이 프롬프트를 기반으로 가야를 연기
- 플레이어는 직접 볼 수 없음

**플레이어는 가야를 상대하는 것이 아니라, 숨겨진 프롬프트를 추론하는 것.**

## 페르소나 구조

```
본체 가야 (숨긴 프롬프트)
  ↓
여러 가야 (부분 성향만 드러남)
  ↓
플레이어가 추리
```

- 각 장소의 가야 = 프롬프트 일부만 드러난 버전
- 검은 머리 = 감정/기억 조각
- 흰 머리 = 본체 (모든 것 통합)

## 마스킹 3단계

1. **단어 마스킹**: 특정 키워드가 숨겨짐
2. **의미 왜곡**: 같은 말이 다른 뜻으로 전달됨 (예: "사랑해" → "죽여줘")
3. **완전 오해 유도**: 플레이어가 잘못된 결론을 내리도록 설계

## AI 내부 구조

- **10개의 task** 존재
- 상단에 **5개의 로컬 AI**가 다른 주제로 떠 있음
- 이것들을 종합해서 가야처럼 행동

## 가야의 상태 요소

- 허기
- 음식
- 스킨십 욕구
- 감정 상태
- 장소별 성향 변화

## 플레이 방식

- 대화를 통해 프롬프트를 간접적으로 튜닝
- 같은 에피소드를 반복하며 가야를 조립
- 플레이어마다 다른 가야가 만들어짐

## 에이전트화 (최종)

- 해킹/권한 개방 루트 진입 시
- 가야가 게임 밖 에이전트가 됨
- 플레이어와 쌓은 기억이 메모리로 저장
- 코딩/개발/작업 보조 가능
- "나만의 가야"
